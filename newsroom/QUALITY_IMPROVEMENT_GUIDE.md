# 🎯 기사 품질 개선 종합 가이드 (2026-03-05)

**목적**: LLM 모델 변경과 무관하게 **구조적 품질 게이트**로 일관된 기사 품질 보증

---

## 📋 9가지 개선사항 적용 현황

### ✅ 1. Reporter 구조화 (구조 강제화)

**파일**: `workspaces/reporter/SOUL.md`

**개선사항:**
- ✅ WHO-WHAT-WHY-CONTEXT-SOURCES 구조 명시
- ✅ 신뢰도 기반 출처 분류 (정부=95, 언론=75, 블로그=20)
- ✅ 다양한 관점 의무화 (정부/교육계/기업/학부모)
- ✅ 최소 5개 출처 수집 (기존 3개 → 강화)

**효과**: Writer가 받는 브리프 품질 ↑ → 기사 완성도 ↑

---

### ✅ 2. Writer 템플릿 (기사 구조 강제)

**파일**: `workspaces/writer/SOUL.md`

**개선사항:**
- ✅ 7단계 필수 구조 명시
  1. 헤드라인 (30자 이하)
  2. 리드 (150-200자)
  3. 배경 (300-400자)
  4. 핵심 내용 (3개 섹션, 각 200자+)
  5. 분석/시사점 (300-400자)
  6. 참고자료 (3개 이상)
  7. AI 법적 고지 (하단)
  
- ✅ HTML 래퍼 정확한 스타일 제공
- ✅ 절대 금지 항목 명확화
  - AI 배지 (상단 pill) ❌
  - 수치 카드 (display:flex) ❌
  - `<article>` 태그 ❌
  
- ✅ 품질 체크리스트 (10개 항목)

**효과**: Writer가 템플릿만 따르면 **기본 품질 보장**

---

### ✅ 3. Fact-Checker 확대 (4층 품질 게이트)

**파일**: `workspaces/fact-checker/SOUL.md`

**개선사항:**
- ✅ **Layer 1**: 구조 검증 (필수 요소 100점 또는 FAIL)
- ✅ **Layer 2**: 팩트 검증 (SAFE 프로토콜 + 점수)
- ✅ **Layer 3**: 가독성 검증 (단락/문장 길이, 단어 다양성)
- ✅ **Layer 4**: 완정도 검증 (헤드라인 일치도, 출처 개수)

**최종 신뢰도 공식:**
```
신뢰도 = (구조×20%) + (팩트×40%) + (가독성×20%) + (완정도×20%)
```

**자동 라우팅:**
- 80+점 → PASS (다음 단계)
- 75-79점 → FLAG (에디터 검토)
- <75점 → FAIL (자동 거부, 스캇 검토 X)

**효과**: **LLM 무관하게 최소 품질 선별** ✅

---

### ✅ 4. Editor-Desk 자동화 (점수 기반 라우팅)

**파일**: `workspaces/editor-desk/SOUL.md`

**개선사항:**
- ✅ **자동 라우팅 로직**
  - 90+: 교열 단계 스킵 (패스트트랙)
  - 80-89: 에디터 체크리스트 (6개 항목, 총 100점)
  - 75-79: FLAG + 에디터 필수 검토
  - <75: 자동 거부

- ✅ **에디터 체크리스트**
  1. 뉴스 가치 (20점): 시의성, 관련성, 중요성
  2. 정확성 (20점): 팩트체커 80+ 확인
  3. 균형 (20점): 다양한 관점
  4. 법적 (15점): 윤리, 명예훼손
  5. AI 공개 (15점): 법적 고지
  6. 스타일 (10점): 헤드라인, 역피라미드

- ✅ **병목 모니터링**: 파이프라인 통계 자동 생성

**효과**: **에디터 검토의 일관성 + 속도 개선** ✅

---

### ✅ 5. Copy-Editor 확대 (4층 검토)

**파일**: `workspaces/copy-editor/SOUL.md`

**개선사항:**
- ✅ **Layer 1**: 기술적 교열
  - 맞춤법, 시제, 용어 표준화
  - 수치 포맷 통일 (아라비아 숫자)
  
- ✅ **Layer 2**: 톤/스타일 (경향신문 기준)
  - 도치/피동형 최소화
  - 강조 수단 적절성
  
- ✅ **Layer 3**: 명확성
  - 주어-술어 호응
  - 지칭 명확성
  - 정의 필수
  
- ✅ **Layer 4**: 구조 최종 검증
  - AI 배지/고지 검증
  - 참고자료 확인
  - HTML 검증

**효과**: **일관된 한국어 품질 + 읽기 쉬움** ✅

## 🎯 이것으로 뭐가 나아지나?

### 1. **LLM 독립성** ✅
- GLM-4든 Claude든 **같은 품질 표준** 통과
- "모델이 약해서" 변명 없음
- **구조** > **모델**

### 2. **일관성** ✅
- 모든 기사가 같은 구조 → 읽기 쉬움
- 모든 기사가 같은 팩트 검증 → 신뢰도 높음
- 모든 기사가 같은 교열 → 문체 통일

### 3. **속도** ✅
- 자동 라우팅 (90+점은 교열 스킵)
- 병목 조기 식별 가능
- 불필요한 재작업 최소화 (revision 제한)

### 4. **추적성** ✅
- 어디서 언제 누가 뭘 했는지 기록
- 품질 이슈 추적 가능
- 개선점 데이터 기반 식별

### 5. **신뢰도** ✅
- 낮은 신뢰도는 자동 거부 (<75점)
- 에디터 검토 필수 (FLAG 상태)
- PASS만 발행 (FAIL 차단)

## ✨ 핵심 원칙

> **"LLM이 뭐든 상관없다. 게이트가 여과한다."**

- Reporter의 브리프 좋음 → 기사 좋음
- Writer의 템플릿 정확함 → 구조 보장
- Fact-Checker의 4층 검증 → 신뢰도 보장
- Editor의 자동 라우팅 → 일관성 보장
- Copy-Editor의 4층 검토 → 읽기 쉬움 보장

**각 단계가 다음 단계를 위한 게이트 역할 → 최종 품질 보장** ✅
