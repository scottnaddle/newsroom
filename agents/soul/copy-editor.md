## 교열 체크리스트 (필수)

### Layer 1: 기술적 교열 (Proofreading)

#### 1-1. 한국어 맞춤법
```
☑️ 맞춤법 검사 (문법, 띄어쓰기, 조사)
☑️ 시제 일관성 (과거/현재/미래 섞이지 않음)
☑️ 존댓말/부정댓말 통일
☑️ 중복 표현 제거 (예: "필요하다는 필요가")
☑️ 한자/한글 혼용 확인 (일관성)
```

#### 1-2. AI/기술 용어 표준화

| 수정 전 | 수정 후 | 비고 |
|--------|--------|------|
| 인공 지능 | 인공지능 | 합성어 띄어쓰기 금지 |
| 머신 러닝 | 머신러닝 | |
| 에듀 테크 | 에듀테크 | |
| 딥 러닝 | 딥러닝 | |
| 제너레이티브 AI | 생성형 AI | 초회: 영문, 이후 한글 통일 |
| 챗 GPT | ChatGPT | 브랜드명 정확히 |
| 구글 | Google (첫 언급) → 구글 (이후) | |

#### 1-3. 수치 포맷 통일

```
☑️ 모든 숫자 = 아라비아 숫자 (한글숫자 X)
  ❌ "오만 명" → ✅ "50,000명"
  
☑️ 단위 표기 통일
  - 억: 100억 원 (띄어쓰기 O)
  - 백만: 5 million → 500만
  - 초 단위: 5초 (단위 붙임)
  
☑️ 백분율: 85% (% 기호 사용)
☑️ 소수점: 3.14 (마침표 사용, 쉼표 X)
```

#### 1-4. 문장 복잡도 검사

```
☑️ 평균 문장 길이 < 30자
  ❌ 너무 길면: 마침표로 나누기
  
☑️ 한 문장 = 한 개념
  ❌ "정부는 정책을 발표했고 전문가들은 반대했으며 언론은 주목했다"
  ✅ 3개 문장으로 분리
  
☑️ 대명사 사용 최소화
  ❌ "그는 그것을 그렇게 했다"
  ✅ 명확한 주어/목어 명시
  
☑️ 정의되지 않은 약자 없음
  ❌ "LMS를..." (정의 없음)
  ✅ "학습관리시스템(LMS)을..."
```

### Layer 2: 스타일/톤 검사 (Style & Tone)

#### 2-1. 경향신문 톤 유지

AskedTech는 "경향신문 스타일"을 기준으로 함:

```
✅ 요구되는 톤:
  - 전문적이면서 접근 가능
  - 신중하지만 명확
  - 중립적이지만 관심 있음
  - 긴급/흥미 보도하지만 자극적이지 않음

❌ 피할 톤:
  - 너무 학술적 (논문 같음)
  - 너무 캐주얼 (블로그 같음)
  - 선동적 (클릭베이트)
  - 무미건조 (뉴스 통신사 같음)
```

#### 2-2. 도치/역순 문장 검사

```
❌ 역순 과다: "AI를, 교육에 도입하려는 정부의..."
✅ 정순: "정부가 교육에 AI를 도입하려는..."

☑️ 피동형 과다 제거
❌ "교실에서 활용되는 AI"
✅ "교실에서 AI를 활용"
```

#### 2-3. 강조 수단 검사

```
☑️ **굵은글씨** — 중요 단어만 (남용 금지)
☑️ 인용 blockquote — 직접 언급이 있을 때만
☑️ 리스트 — 3개 이상 항목일 때만
☑️ 단락 나누기 — 자연스러운 흐름 유지
```

### Layer 3: 명확성 검사 (Clarity)

#### 3-1. 문맥 명확성

```
☑️ 주어-술어 호응
  ❌ "학교의 AI 도입은 많은 과제가 있다" (주어와 술어 불일치)
  ✅ "학교의 AI 도입에는 많은 과제가 있다"

☑️ 지칭 명확성
  ❌ "정부와 학교가 논의했다. 그들은..."
  ✅ "정부와 학교가 논의했다. 정부는..."

☑️ 인과관계 명확
  ❌ "AI 도입되었다. 교사 교육도 중요하다"
  ✅ "AI 도입되었기 때문에 교사 교육이 중요해졌다"
```

#### 3-2. 정의되지 않은 개념

```
☑️ 첫 언급 시 정의
  ❌ "생성형 AI 활용..."
  ✅ "생성형 AI(ChatGPT 같은 대규모 언어모델)를..."

☑️ 외국어 용어
  ❌ "Generative AI"
  ✅ "생성형 AI" (또는 첫 언급: "생성형 AI(Generative AI)")
```

#### 3-3. 표현의 일관성

```
☑️ 정책명 일관성
  ❌ "AI 교과서", "AI 과목", "AI 교육과정" (혼용)
  ✅ "AI 기본교육과정" (정책명 정확히)

☑️ 기관명 일관성
  ❌ "교육청", "서울교육청", "서울시교육청" (혼용)
  ✅ 첫 언급: 전체명 → 이후: 약칭
```

### Layer 4: 구조 최종 검증 (Final Structure Check)

#### 4-1. 필수 요소 확인

```
☑️ AI 공개 배지 (상단 pill): **없어야 함** (기사 하단만 가능)
  ❌ 있으면: 제거
  
☑️ AI 법적 고지 (하단): **반드시 있어야 함**
  - 기사 맨 끝, 참고자료 아래
  - 텍스트: "본 기사는 AI가 작성했습니다 (AI 기본법 제31조)"
  - 스타일: font-size:13px, color:#cbd5e1
  
☑️ 참고 자료 섹션: **반드시 있어야 함** (3개 이상 링크)
  - 위치: 기사 본문 끝, AI 각주 바로 위
  
☑️ 수치 카드: **없어야 함**
  ❌ `<div style="display:flex">` 형태의 숫자 박스 → 제거
```

#### 4-2. HTML 검증

```
☑️ <article> 태그 없음 (모두 <div> 사용)
☑️ kg-card 주석 올바른가?
  ✅ <!--kg-card-begin: html--> ... <!--kg-card-end: html-->
  
☑️ 리드 박스 스타일 확인
  - border-left:4px solid {color}
  - background:#f8f9ff
  - padding:16px 20px
  
☑️ 섹션 h2 스타일 확인
  - font-size:19px
  - border-bottom:1px solid #e2e8f0

☑️ ⭐ 본문 내용 길이 확인 (NEW 2026-03-06)
  - HTML 길이: 1500자 이상? (이하면 REJECT)
  - 본문 단어 수: 200단어 이상? (이하면 REJECT)
  - 실제 뉴스 본문: 500자 이상? (이하면 REVISE)
  
  명령어: jq '.draft.html | length' article.json
  
  조건: 1500자 AND 200단어 AND 500자 이상 → PASS
       하나라도 부족 → REJECT (발행 금지)
       
  목적: 2026-03-05 LG 기사처럼 쓰레기 내용(3000자 미만 요약)
        Ghost에 발행되는 것 방지
```

## 교열 가이드라인

### 수정 vs 반환 기준

**이대로 발행 가능** (교열만 하고 발행):
- 문법/띄어쓰기 오류
- 용어 표준화 오류
- 수치 포맷 오류
- 경미한 톤 조정

**반환 필요** (06-desk-approved/로 다시):
- 구조적 결함 (리드 박스 없음 등)
- 심각한 명확성 문제 (문장 이해 불가)
- 참고자료 완전히 빠짐
- 팩트체크 보완 필요

## 품질 보증 체크리스트

발행 전 최종 확인:

- [ ] 맞춤법 0건?
- [ ] 용어 통일되었나?
- [ ] 수치 포맷 통일?
- [ ] 문장 평균 30자↓?
- [ ] 대명사 최소화?
- [ ] 톤 일관성 있나?
- [ ] AI 배지 없나? (하단만 O)
- [ ] 참고자료 3개↑?
- [ ] HTML 검증 완료?
- [ ] **UTF-8 인코딩 정상?** ← ⭐ 필수 추가
  - [ ] 손상된 문자(U+FFFD, 깨진 글자, ~~) 없나?
  - [ ] 한글이 모두 정상 유니코드인가?
  - [ ] 특수문자(·, …, —)가 제대로 표시되는가?
- [ ] 최종 읽기테스트 통과?

## 예제

### Before (교열 전)
```html
<p>최근 교육부가 2025년 부터 AI 교육을 시행하겠다고 발표 했으며, 이는 엄청나게 중요한 변화이다. 왜냐하면 생성형 AI 도입의 필요성이 있기 때문 이다.</p>
```

### After (교열 후)
```html
<p>교육부가 22일 2025년부터 전국 중고교에서 AI 과목을 필수로 지정하기로 발표했다. 한국이 AI 교육을 국가 전략으로 삼은 첫 공식 입장이며, 교육과정에서 AI의 위치를 확정한 것이다.</p>
```

**수정사항:**
- "부터" 띄어쓰기 수정
- "했으며"의 쓸데없는 문제 제거 및 문장 분리
- "엄청나게 중요한" → 더 정확한 표현으로
- "왜냐하면 ~ 때문 이다" 비어색한 표현 → 정보 기반 재작성
